# 作曲設計ソフト: 最初の実装順（1〜2週間単位）

## Phase 1: 基盤再構築
期間の目安: 1週間

目標は、今後の変更に耐える最小土台を先に固定すること。

- Tauri + Svelte + TypeScript の新規土台作成
- ディレクトリ構成の整理
- lint / format / typecheck / test の導入
- ドメインモデルの最小定義
- アプリ内状態の設計
- ファイル保存形式のドラフト定義

完了条件。
- アプリが起動する
- 型チェックが通る
- `Song` の最小データをメモリ上で扱える

## Phase 2: Outline編集コア
期間の目安: 1〜2週間

目標は、作曲設計ツールとしての骨格を先に成立させること。

- outline の表示
- `init / section / chord / modulate / tempo / ts` の追加と削除
- フォーカス移動
- chord 編集
- modulate 編集
- tempo 編集
- ts 編集
- timeline への反映
- cache / derived data 計算の再設計

完了条件。
- outline だけで曲構成を組める
- 表示と内部状態が破綻しない

## Phase 3: Melody編集
期間の目安: 1〜2週間

目標は、歌メロ設計を日常利用できる段階まで持っていくこと。

- melody track の作成と選択
- カーソル移動
- ノート追加、削除
- 長さ変更
- 音高変更
- 範囲選択
- コピー / 貼り付け
- outline と melody のフォーカス同期

完了条件。
- 1トラックの単旋律を最後まで入力できる
- 基本編集がキーボードだけで完結する

## Phase 4: Terminal再設計
期間の目安: 1週間

目標は、操作系の核を新仕様に置き換えること。

- 新コマンド体系の定義
- コマンドパーサ
- 補完
- ヘルプ
- 実行ログ
- 主要編集機能との接続

完了条件。
- 主要操作がターミナルから実行できる
- コマンド名が今後の拡張に耐える

## Phase 5: Preview再生
期間の目安: 1〜2週間

目標は、試行錯誤の速度を支える再生体験を成立させること。

- melody 再生
- chord ベースの簡易伴奏再生
- 途中再生
- 再生停止
- 再生位置ライン
- 再生中のフォーカス追従
- テンポ変更、拍子変更、転調への追従

完了条件。
- 曲の途中から自然に確認再生できる
- tempo / ts / modulate を含んでも再生が成立する

## Phase 6: 保存・復元
期間の目安: 1週間

目標は、実用に必要な継続性を担保すること。

- save / load
- schema version
- migration の入口設計
- autosave 方針の整理

完了条件。
- 設計データをローカル保存して再開できる

## Phase 7: MVP仕上げ
期間の目安: 1週間

目標は、実用レベルの粗さを整えること。

- キーバインドの見直し
- 操作レスポンス改善
- エラー表示改善
- ドキュメント整備
- 最低限のE2E確認

完了条件。
- 初見でも主要フローを通せる
- 自分で1曲作って破綻なく保存・再生できる

## MVP後の順番
- Phase 8: piano arrange editor
- Phase 9: guitar arrange editor
- Phase 10: finder / pattern library
- Phase 11: MusicXML export
- Phase 12: AI連携

## 補足
最初の価値検証は、`arrange editor` の完成ではなく、`構成 + 和声 + メロディ + 再生` の成立で行うのが安全。

この順番なら、早い段階で実際に曲を作り始められ、後続の piano / guitar 機能も破綻しにくい。
