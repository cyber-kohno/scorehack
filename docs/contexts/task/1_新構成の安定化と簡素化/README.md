# 1_新構成の安定化と簡素化

## 目的
`src/system` を新構成へ移し切った直後の状態を安定化させ、今後の機能追加や保守で迷いにくい土台へ整える。

## このフォルダに残すもの
- タスク全体の計画
- 日付ごとの作業ログ
- 回帰確認チェックリスト
- 個別論点の整理メモ

## 優先事項
1. 挙動の安全網を作る
2. `root-store` 依存の形だけ残っている部分を整理する
3. selector / helper の `$store` 依存を減らす
4. 型の置き場ルールを統一する
5. 薄い facade / helper を減らす
6. 複雑ファイルの責務をもう一段整理する
7. コメント保全と編集ルールを運用として固定する

## 当面の進め方
1. 回帰確認チェックリストを基準にする
2. `StoreProps` / `StoreUtil` / `lastStore` の棚卸しを行う
3. 互換名残を分類して、低リスクな箇所から整理する
4. その後に型配置ルールと複雑ファイルの整理へ進む
