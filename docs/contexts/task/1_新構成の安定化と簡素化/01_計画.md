# 計画

## タスクの目的
新構成へ移し切った直後の状態を安定化させ、理解しやすく保守しやすい形に整える。

## 優先順位
1. 挙動の安全網を作る
2. `root-store` 依存の形だけ残っている部分を整理する
3. selector / helper の `$store` 依存を減らす
4. 型の置き場ルールを統一する
5. 薄い facade / helper を減らす
6. 複雑ファイルの責務をもう一段整理する
7. コメント保全と編集ルールを運用として固定する

## 直近の実施順
1. 回帰確認チェックリストを基準として固定する
2. `StoreProps` / `StoreUtil` / `lastStore` の棚卸しを行う
3. 互換名残を分類して、低リスクな箇所から順に整理する
4. その後に型配置ルール、複雑ファイル整理へ進む

## 今やること
- `root-store` 依存整理
- 具体的には `StoreProps` / `StoreUtil` / `lastStore` の使用箇所を分類する
- その中から、実際には `root-store` が不要な helper / session を先に減らす
