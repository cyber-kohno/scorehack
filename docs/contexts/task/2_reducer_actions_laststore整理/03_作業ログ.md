# 2026-04-23 melody-reducer の初回棚卸し
- `10_melody-reducer責務棚卸し.md` を追加した

`melody-reducer` の公開 API を
- mutation
- selector/accessor
- orchestration

に分類した。
最初の分離候補は、
- `getCurrScoreTrack`
- `getFocusRange`

の selector/accessor 側から見るのが安全、という判断を残した。

# 2026-04-23 melody-reducer から selector を分離
- `melody-selectors.ts` を追加した
- `melody-reducer.ts` から
  - `getCurrScoreTrack`
  - `getFocusRange`
  を外した
- `melody-actions.ts` は互換 API を保ちつつ、新しい selector helper を使う形に更新した

まずは reducer の責務を削り、caller の崩れを最小化する方針で進めた。

# 2026-04-23 melody-reducer から addNote 系 mutation を分離
- `melody-mutations.ts` を追加した
- `melody-reducer.ts` から
  - `addNote`
  - `addNoteFromCursor`
  を外した
- `melody-actions.ts` で
  - selector
  - cursor copy
  - 純粋 mutation
  を組み合わせて互換 API を維持する形に更新した

これで `melody-reducer` は、更新の中でも局所的な mutation を一段手放した。

# 2026-04-23 melody-reducer から judgeOverlap を分離
- `melody-selectors.ts` に overlap 判定を追加した
- `melody-actions.ts` で overlap state 更新を受け持つ形に変えた
- `melody-reducer.ts` から `judgeOverlap` を外した

「現在のノート群を読んで判定する責務」と「state を更新する責務」を分けることで、
reducer を orchestration 寄りに寄せた。

# 2026-04-23 melody-reducer から track 切替を分離
- `melody-track-actions.ts` を追加した
- `changeScoreTrack` を
  - track index 更新
  - cursor 同期
  に分け、`melody-actions.ts` 側で組み立てる形にした
- `melody-reducer.ts` から `changeScoreTrack` を外した

これで reducer に残る責務は、さらに focus / cursor 同期寄りに絞られた。

# 2026-04-23 melody-reducer から focus 系 orchestration を追加分離

# 2026-04-24 token 軸の再構成方針を固定
- `12_token軸の再構成方針.md` を追加した
- 今後の実施ステップを
  1. token 不要な関数を factory から外す
  2. token が必要な関数群を factory 化する
  3. token 不要な関数群を namespace でまとめるか判断する
  に固定した

現時点では、3 の namespace 化は優先度を下げ、1 と 2 を先に進める方針とした。

# 2026-04-24 token 不要な addNote 系を factory から外した

  を直接使う形に変更した
- `melody-actions.ts` から
  - `addNote`
  - `addNoteFromCursor`
  を外した

これで `createMelodyActions()` は、token 不要な純粋 mutation を持たない形に一歩近づいた。

# 2026-04-24 token 必要な selector を factory 化
- `melody-selectors.ts` を `createMelodySelectors(rootStoreToken)` 形式に変更した
- `melody-actions.ts` は selector を top-level 関数で直接呼ばず、selector factory から受け取る形に変更した

これで `token 必要な責務は factory` という方針の最初の実例が入った。
