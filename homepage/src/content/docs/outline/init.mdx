---
title: Init
description: 曲の初期設定を定義するブロック
---

Init は、曲の初期状態を定義するためのブロックです。

Outline の先頭に存在し、曲全体の時間軸や音楽理論上の前提を決めます。  
メロディ、コード、アレンジ、エクスポートは、この Init で定義された情報を基準に動作します。

## Init の責務

Init は主に次の情報を担います。

- 曲の初期キー
- メジャー / マイナーなどのスケール
- 初期テンポ
- 初期拍子
- 以降の Chord や Event が参照する基準情報

Init は曲の出発点です。  
曲の途中でキー、テンポ、拍子を変えたい場合は、Init を変更するのではなく Event を配置します。

## Outline 上での位置づけ

Init は通常、Outline の最初のブロックとして扱います。

Section や Chord は、Init によって定義された初期状態の上に積み上がります。  
たとえば Init が `C major` であれば、後続の度数コードは C major を基準に解釈されます。

## Melody への影響

Melody では、Init のキーとスケールがメロディ入力の基準になります。

たとえば、スケール内の音かどうか、コード構成音が調性内かどうかといった判定は、Init から始まる調性情報をもとに行われます。

途中で転調イベントがある場合は、その位置以降のメロディ判定に転調後の調性が使われます。

## Chord への影響

Chord は、Init の調性をもとに度数コードを解釈します。

たとえば、同じ `I` でも Init が `C major` なら C、`G major` なら G として扱われます。  
これにより、曲全体を移調したい場合でも、度数ベースのコード進行を保ちやすくなります。

## Arrange への影響

Arrange は、Chord の解釈結果をもとに伴奏やボイシングを作ります。

そのため Init の調性は、アレンジで扱う構成音、スケール判定、候補表示にも影響します。

## Export への影響

MusicXML などのエクスポートでは、Init の拍子、テンポ、調性が出力の初期値として使われます。

外部の譜面ソフトや歌声合成ツールに渡す時、Init の情報は曲の前提として重要です。

## コピーと削除

Init は曲全体の前提になるため、通常のブロックと同じようには扱いません。

たとえば、Outline のコピー対象に Init が含まれる場合はコピーできません。  
これは、貼り付け先に複数の Init が混ざると、曲の前提が不明確になるためです。

## Event との違い

Init は「曲の開始時点の状態」を定義します。

一方、Event は「曲の途中で状態を変更する」ためのブロックです。

| ブロック | 役割 |
| --- | --- |
| Init | 曲の初期状態を決める |
| Event | 曲の途中で状態を変える |

たとえば、曲の最初から `120 BPM` にしたい場合は Init、途中から `140 BPM` にしたい場合は Tempo Event として扱います。

## Keyboard

Init にフォーカスしている時は、曲全体の初期条件をキーホールドで変更できます。

  <thead>
    <tr><th>キー</th><th>操作</th></tr>
  </thead>
  <tbody>
    <tr><td><kbd>D</kbd> + <kbd>↑</kbd> / <kbd>↓</kbd></td><td>キーを上下に変更します。</td></tr>
    <tr><td><kbd>D</kbd> + <kbd>←</kbd> / <kbd>→</kbd></td><td>major / minor を切り替えます。</td></tr>

    <tr><td><kbd>C</kbd> + <kbd>↑</kbd> / <kbd>↓</kbd></td><td>feel を変更します。</td></tr>
    <tr><td><kbd>F</kbd> + <kbd>↑</kbd> / <kbd>↓</kbd></td><td>テンポを変更します。</td></tr>
