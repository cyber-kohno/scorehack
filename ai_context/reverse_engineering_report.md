# ScoreHack Reverse Engineering Report

現存システム (`svelte_app`) の解析に基づき、アプリの概要・機能・技術スタック、および再設計に向けた整理を行いました。

## 1. アプリケーション概要
ScoreHackは、単なるDAW（録音・編集ソフト）ではなく、**「作曲の設計」を支援する作曲補助ツール**です。
楽曲の構造（コード進行、テンポ、転調、セクション）を「アウトライン」として定義し、それに基づいてメロディやバッキング（アレンジ）を構築していくアプローチを特徴としています。

---

## 2. 主要機能の整理

### A. アウトライン編集（Harmonizeモード）
楽曲のスケルトンを構築する中心機能です。
- **セクション管理**: イントロ、サビ、Aメロなどのセクションを定義。
- **度数（Degree）によるコード入力**: 「C, G」といった固定名ではなく「I, V」といったディグリーでの入力を基本とし、キー（調）に依存しない設計が可能。
- **転調（Modulation）管理**: 属調、同主調、平行調などの転調を明示的に定義し、以降のコード入力を自動追従。
- **テンポ・拍子（TS）変更**: 楽曲途中の動的な変化に対応。

### B. メロディ編集（Melodyモード）
ピアノロール形式のメロディ入力機能。
- **スケール追従**: 現在のコードやキーに基づいたスケール内の音を視覚的または入力時に補助する仕組み。
- **直感的な操作**: キーボードによるノート入力、移動、長さの調節。

### C. アレンジ（バッキング）システム
コード進行から自動的に伴奏を生成する強力なエンジン。
- **バッキングパターン**: リズムとタイミング（打鍵、ペダル等）のテンプレート。
- **ボイシングパターン**: コード構成音（Root, 3rd, 5th等）のどれをどのオクターブで鳴らすかの定義。
- **自動生成**: 上記2つの組み合わせを、現在のコード構造（Music Theoryエンジン）に適用して実際の演奏ノーツを生成。

### D. 再生・出力システム
- **SoundFont再生**: ブラウザ上で `soundfont-player` を用い、多様な音色でのプレビューが可能。
- **MIDIエクスポート**: `midi-writer-js` を利用した作曲データのMIDI書き出し。
- **プロジェクト保存**: Gzip圧縮されたJSON（`.sch`）形式での独自フォーマット保存。

### E. UI/UX 特徴
- **ハイブリッドUI**: GUI操作に加え、`t`キーで呼び出す **Terminal（CUI）** によるコマンド操作が可能。
- **キーボード重視**: 特殊な「Holdキー（E, D, F, C, X, G等）」を組み合わせたショートカットにより、マウスレスな高速編集を志向。

## 3. 技術アーキテクチャ

- **Frontend**: Svelte 5 + TypeScript + Vite
- **State Management**: 
    - カスタム `writable` ストアを用いた独自のステート管理。
    - [Reducer](file:///c:/app/git/scorehack/svelte_app/src/system/util/preview/previewUtil.ts#137-155) パターンを取り入れ、変更（commit）後にキャッシュの再計算（calculate）を走らせる堅牢な設計。
- **Logic**:
    - `MusicTheory.ts`: 度数、コード、スケール、インターバル計算を司る中枢エンジン。
    - `ReducerCache.ts`: 時間軸（ミリ秒・拍）とUI座標の変換、演奏データのキャッシュ化を担当。

## 4. Tauriへの移植とAI補助機能の統合（再設計プラン）

### Tauri への移行メリット
1. **パフォーマンス**: 複雑な計算（アレンジ生成、MIDI処理）を Rust 側で実行可能。
2. **ネイティブ連携**: Web File System Access API に依存せず、より安定したファイル入出力。
3. **MIDIデバイス連携**: Rust の `midir` ライブラリなどを用い、外部MIDI機器との高度な通信が可能に。

### AI補助機能の組み込み案（要件案）
AIを「作曲の設計士」のパートナーとして配置するための新機能案です。

- **AIコード提案**: セクションの文脈や指定したジャンルに基づき、適切なコード進行を提案（アウトライン編集の拡張）。
- **AIアレンジ生成**: 「ジャズ風」「エレクトロ風」といった言葉の指示から、バッキング/ボイシングパターンをAIが動的に生成。
- **メロディ・コンプリート**: 数小節のメロディを入力すると、その続きを現在のコード進行に合わせてAIが提案。
- **自然言語コマンド（Terminal AI）**: ターミナルに「ここから4小節、切ない雰囲気のサビを追加して」と入力すると、アウトラインが自動構築される。
- **理論的アドバイス**: 「この転調は急すぎるかもしれない」等の音楽理論に基づいたフィードバックをAIが提供。

---

## 5. 次のステップへの提案

1. **データ構造のRust化**: 音楽理論エンジン（`MusicTheory.ts`）のコア部分を Rust に移植し、データ操作を高速化。
2. **AI API の統合**: Gemini などのLLMを Tauri (Rust) バックエンド経由で呼び出す基盤の構築。
3. **UIデザインのモダン化**: Tauri + Svelte の強みを活かし、よりデスクトップアプリらしい、リッチなアニメーションとレスポンスの向上。
